# Bifrost

SQL Server data migration tool for .NET 10. Named after the rainbow bridge connecting realms in Norse mythology.

## Projects

- **Bifrost.Core** — shared migration logic (models, database, export, import, direct, restructure)
- **Bifrost.CLI** — command line interface
- **Bifrost.GUI** — Avalonia cross-platform desktop app

## Build

```bash
dotnet build
```

## CLI Usage

### export
Exports tables from the source database to `.sql` files. Use when source and target are on different VPNs.

```bash
bifrost export
bifrost export --config config.json --output ./output
```

### import
Imports `.sql` files generated by `export` into the target database.

```bash
bifrost import
bifrost import --config config.json --output ./output
```

### direct
Migrates directly from source to target with no intermediate files. Use when both servers are reachable on the same network.

```bash
bifrost direct
bifrost direct --config config.json
bifrost direct --config config.json --bulk
```

The `--bulk` flag uses `SqlBulkCopy` for significantly faster transfers.

### restructure
Moves tenant tables into named schemas and strips the tenant ID suffix from table names. Optionally creates compatibility views using the old table name.

```bash
bifrost restructure
bifrost restructure --config config.json
```

## Publish as single exe

```bash
# Windows
dotnet publish Bifrost.CLI -r win-x64 -c Release
dotnet publish Bifrost.GUI -r win-x64 -c Release

# Linux
dotnet publish Bifrost.CLI -r linux-x64 -c Release
dotnet publish Bifrost.GUI -r linux-x64 -c Release
```

## Config file format

All commands share a single config file containing both connection details and database/tenant definitions.

```json
{
  "source": {
    "server": "192.168.1.100",
    "port": 1433,
    "username": "sa",
    "password": "yourpassword",
    "encrypt": false,
    "trustServerCertificate": true
  },
  "target": {
    "server": "192.168.2.100",
    "port": 1433,
    "username": "sa",
    "password": "yourpassword",
    "encrypt": false,
    "trustServerCertificate": true
  },
  "databases": [
    {
      "comment": "Tenant filter example",
      "sourceDatabase": "MainAppDB",
      "targetDatabase": "MainAppDB_Staging",
      "tableFilter": "tenant",
      "tenantId": "142",
      "overrides": [
        { "name": "dbo.HugeTable_142", "where": "CreatedAt >= '2024-01-01'" },
        { "name": "dbo.BigTable_142", "query": "SELECT TOP 1000 * FROM [dbo].[BigTable_142] ORDER BY CreatedAt DESC" },
        { "name": "dbo.UnwantedTable_142", "ignore": true }
      ]
    },
    {
      "comment": "Explicit table list",
      "sourceDatabase": "ReportingDB",
      "targetDatabase": "ReportingDB_Staging",
      "tableFilter": "explicit",
      "tables": [
        "dbo.SmallTable",
        { "name": "dbo.HugeTable", "where": "CreatedAt >= '2024-01-01'" },
        { "name": "dbo.UnwantedTable", "ignore": true }
      ]
    },
    {
      "comment": "All tables",
      "sourceDatabase": "LookupDB",
      "targetDatabase": "LookupDB_Staging",
      "tableFilter": "all",
      "overrides": [
        { "name": "dbo.LargeAuditLog", "ignore": true }
      ]
    }
  ],
  "tenants": [
    {
      "tenantId": "142",
      "schema": "WarehouseCo",
      "database": "MainAppDB_Staging",
      "createCompatibilityViews": true
    },
    {
      "tenantId": "",
      "schema": "WarehouseCo",
      "database": "MainAppDB_Staging",
      "sourceSchema": "test",
      "createCompatibilityViews": false
    }
  ]
}
```

## tableFilter options

| Value | Description |
|-------|-------------|
| `tenant` | All tables ending with `tenantId` |
| `explicit` | Only the listed tables |
| `all` | All tables in the database |

## Per-table override options

| Field | Description |
|-------|-------------|
| `ignore: true` | Skip this table |
| `where: "..."` | Filter rows with a WHERE clause |
| `query: "..."` | Use a fully custom SELECT query |

## Tenant restructure options

| Field | Description |
|-------|-------------|
| `tenantId` | Suffix to match and strip from table names (e.g. `142`) |
| `schema` | Target schema to move tables into |
| `database` | Database to restructure (uses target connection) |
| `sourceSchema` | If set, moves all tables from this schema instead of matching by suffix |
| `createCompatibilityViews` | Creates views in the original schema using the old table name |

## GUI

The GUI stores named configs in `bifrost-configs.json` next to the executable. Configs can be created, edited, and deleted from within the app.
