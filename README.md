# Bifrost

SQL Server data migration tool for .NET 10. Named after the rainbow bridge connecting realms in Norse mythology.

## Projects

- **Bifrost.Core** — shared migration logic (models, database, export, import, direct, restructure, retry, history)
- **Bifrost.CLI** — command line interface
- **Bifrost.GUI** — Avalonia cross-platform desktop app
- **Bifrost.Tests** — xUnit unit tests

## Build

```bash
dotnet build
dotnet test
```

## Releases

Releases are built automatically via GitHub Actions when a version tag is pushed:

```bash
git tag v1.0
git push origin v1.0
```

This runs all tests, publishes self-contained single-file executables for Windows x64 and Linux x64, and attaches them as zips to a GitHub Release. Tags containing a hyphen (e.g. `v1.0-beta`) are marked as pre-releases.

To build manually:

```bash
dotnet publish Bifrost.CLI -r win-x64 -c Release --self-contained -p:PublishSingleFile=true
dotnet publish Bifrost.GUI -r win-x64 -c Release --self-contained -p:PublishSingleFile=true

dotnet publish Bifrost.CLI -r linux-x64 -c Release --self-contained -p:PublishSingleFile=true
dotnet publish Bifrost.GUI -r linux-x64 -c Release --self-contained -p:PublishSingleFile=true
```

## CLI Usage

All commands accept either `--config` pointing to a JSON file, or inline connection arguments. `restructure` and `status` require `--config`.

### export
Exports tables from the source database to `.sql` files.

```bash
bifrost export --config config.json
bifrost export --config config.json --output ./output --dry-run
bifrost export --source-server 192.168.1.100 --source-database AppDB --target-database AppDB_Staging
```

### import
Imports `.sql` files generated by `export` into the target database. Each file is wrapped in a transaction — a failure rolls back that file cleanly.

```bash
bifrost import --config config.json
bifrost import --config config.json --output ./output --dry-run
```

### direct
Migrates directly from source to target with no intermediate files.

```bash
bifrost direct --config config.json
bifrost direct --config config.json --bulk
bifrost direct --config config.json --dry-run
bifrost direct --config config.json --bulk --drop-and-create
bifrost direct --config config.json --append-only

# Inline — single table, same server
bifrost direct --source-server 192.168.1.100 --source-user sa --source-pass secret \
               --source-database AppDB --target-database ArchiveDB \
               --source-table dbo.Orders --target-table dbo.OrdersArchive \
               --bulk --drop-and-create

# With a WHERE clause
bifrost direct --source-server 192.168.1.100 --source-user sa --source-pass secret \
               --source-database AppDB --target-database ArchiveDB \
               --source-table dbo.Orders --where "CreatedAt >= '2024-01-01'" --bulk

# With a custom query
bifrost direct --source-server 192.168.1.100 --source-user sa --source-pass secret \
               --source-database AppDB --target-database ArchiveDB \
               --source-table dbo.Orders \
               --query "SELECT TOP 1000 * FROM [dbo].[Orders] ORDER BY CreatedAt DESC" --bulk

# Without recording history
bifrost direct --config config.json --bulk --no-history

# Inline — different servers
bifrost direct --source-server 192.168.1.100 --source-user sa --source-pass secret \
               --target-server 192.168.2.100 --target-user sa --target-pass secret \
               --source-database AppDB --target-database AppDB_Staging \
               --bulk
```

### restructure
Moves tenant tables into named schemas and strips the tenant ID suffix. Detects naming conflicts before starting and rolls back if a failure occurs mid-run.

```bash
bifrost restructure --config config.json
```

### status
Connects to both source and target and reports table counts per database, plus the last 5 run history entries.

```bash
bifrost status --config config.json
```

## Inline connection options

| Option | Description |
|--------|-------------|
| `--source-server` | Source server IP or hostname |
| `--source-port` | Source port (default 1433) |
| `--source-user` | Source username |
| `--source-pass` | Source password |
| `--source-database` | Source database |
| `--target-server` | Target server (defaults to source server for same-instance transfers) |
| `--target-port` | Target port (default 1433) |
| `--target-user` | Target username (defaults to source username) |
| `--target-pass` | Target password (defaults to source password) |
| `--target-database` | Target database |
| `--table-filter` | `all`, `tenant`, or `explicit` (default `all`) |
| `--tenant-id` | Tenant ID when using `--table-filter tenant` |
| `--source-table` | Single source table e.g. `dbo.Orders` — implies `explicit` filter |
| `--target-table` | Target table name override e.g. `dbo.OrdersArchive` |
| `--bulk` | Use SqlBulkCopy (significantly faster) |
| `--drop-and-create` | Drop and recreate target tables |
| `--append-only` | Insert without deleting existing rows |
| `--dry-run` | Preview without making changes |

## tableFilter options

| Value | Description |
|-------|-------------|
| `tenant` | All tables ending with `_tenantId` or `tenantId` |
| `explicit` | Only the listed tables |
| `all` | All tables in the database |

## Per-table options (overrides and explicit tables)

| Field | Description |
|-------|-------------|
| `name` | Source table name including schema e.g. `dbo.Orders` |
| `targetName` | Target table name override e.g. `dbo.OrdersArchive` (schema optional, defaults to source schema) |
| `ignore: true` | Skip this table |
| `where: "..."` | Filter rows with a WHERE clause |
| `query: "..."` | Use a fully custom SELECT query |

## Database-level options

| Field | Default | Description |
|-------|---------|-------------|
| `dropAndCreate` | `false` | Drop and recreate target tables before inserting |
| `appendOnly` | `false` | Insert without deleting existing rows |

Note: `dropAndCreate` and `appendOnly` are mutually exclusive. If the target table does not exist it will always be created regardless of these settings.

## Tenant restructure options

| Field | Description |
|-------|-------------|
| `tenantId` | Suffix to match and strip from table names |
| `schema` | Target schema to move tables into |
| `database` | Database to restructure (uses target connection) |
| `sourceSchema` | If set, moves all tables from this schema as-is without suffix matching or renaming |
| `createCompatibilityViews` | Creates views in the original schema pointing to the new location |

## Reliability

- **Retry** — failed tables are retried up to 3 times with a 2 second delay
- **Transactions** — each imported `.sql` file runs inside a transaction and is rolled back cleanly on failure
- **Restructure rollback** — if a table fails mid-restructure, all completed moves for that tenant are reversed
- **Conflict detection** — restructure scans for naming collisions before starting and aborts if any are found


## GUI

Named configs are stored in `bifrost-configs.json` next to the executable. Run history is stored in `bifrost-history.json`.

- **Import config** — load a `.json` file and save it as a named config (`↑` button)
- **Export config** — save a named config to a standalone `.json` file (`↓` button per config)
- **Dry Run** — checkbox to preview an operation without writing anything
- **SqlBulkCopy** — checkbox to enable bulk mode for direct migration
- **Drop and recreate** — per-database checkbox to drop and recreate target tables
- **Append only** — per-database checkbox to insert without clearing existing rows
- **Table overrides** — per-database editor for tenant/all filters to add WHERE clauses, custom queries, target name renames, or ignore specific tables
- **Explicit tables** — per-database editor for explicit filter with the same options
- **Progress bar** — shows per-table progress during any operation
- **Recent runs** — last 10 runs shown in the sidebar; click any run to view its saved log
- **Run history** — full history window with success/fail filter, checkboxes to select runs, and a delete button; opens via the "All" button next to recent runs
- **Tray icon** — closing the window hides to tray; quit from the tray menu to exit fully

## Using Bifrost from another project

If you need to trigger Bifrost from a project that can't reference `Bifrost.Core` directly (e.g. .NET Framework), copy `integrations/BifrostRunner.cs` into your project and use it to shell out to the CLI executable:

```csharp
var bifrost = new BifrostRunner(
    Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "bifrost.exe"));

// Single table transfer with rename
var result = bifrost.DirectTable(
    sourceServer:   "192.168.1.100",
    sourceDatabase: "AppDB",
    sourceTable:    "dbo.Orders",
    targetDatabase: "ArchiveDB",
    targetTable:    "dbo.OrdersArchive",
    sourceUser:     "sa",
    sourcePass:     "secret",
    bulk:           true,
    dropAndCreate:  true
);

if (!result.Success)
    throw new Exception(result.Output + result.Error);
```

`BifrostRunner` automatically answers the confirmation prompt, captures stdout/stderr, and returns a `BifrostResult` with `Success`, `ExitCode`, `Output`, and `Error`.
